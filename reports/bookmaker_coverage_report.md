## 2. Прямая проверка каждого источника

### The Odds API
Запросил официальный список БК по всем поддерживаемым регионам (`us`, `us2`, `uk`, `eu`, `fr`, `se`, `au` — это все регионы, которые API поддерживает в принципе, не только используемый `eu`).

**Результат: ни «Зенит», ни Melbet, ни Fonbet, ни Winline в списке нет ни в одном регионе.** The Odds API фокусируется на лицензированных БК Европы/Великобритании/США/Австралии/Франции/Швеции.

### API-Football (`/odds/bookmakers`)
Реальный вызов вернул каталог из 33 БК. В каталоге **есть Fonbet (id 33)**. Melbet, Winline, «Зенит» — отсутствуют.

Дальше проверил, появляется ли Fonbet реально в ответах `/odds` для текущих матчей (а не только в справочнике имён): выгрузил коэффициенты по всем матчам за 3 доступных на бесплатном плане даты (2026-07-14…07-16, 50 матчей). **Fonbet не встретился ни в одном из 50 матчей** — присутствует в справочнике, но фактически не поставляет данные в рамках доступного покрытия/плана. Это нужно перепроверять по мере наполнения линии ближе к матчу, но на сегодня это ноль реальных данных, а не 4 из 4 БК.

## 4. Что можно подключить для недостающих БК

Искал специализированные источники именно под «Зенит»/Melbet/Fonbet/Winline (без парсинга сайтов, только официальные API-провайдеры):

- **odds-api.io** — агрегатор 260+ БК, у него есть отдельный официальный продукт **MelBet Sportsbook API** (REST, WebSocket, id и документация подтверждены). Тарифы: Free — 2 БК, 100 запросов/час (для проверки, не для продакшена); Starter — £99/мес, 5 БК, 5000 запросов/час; Growth — £179/мес, 10 БК. Это закрывает **только Melbet** из четырёх — прямых продуктов «Зенит», Fonbet, Winline у него нет (проверил по прямым ссылкам — 404).
- Fonbet, Winline, «Зенит» — специфически российские БК. У них нет публичного лицензированного API у крупных международных агрегаторов (The Odds API, API-Football, odds-api.io, и в целом у провайдеров, ориентированных на регулируемые рынки ЕС/США/Великобритании/Австралии) — вероятная причина: лицензирование ограничено российским рынком, международные агрегаторы туда не идут.
- Единственный способ получить их коэффициенты без официального API — прямой парсинг сайтов/приложений этих БК, который явно запрещён в задаче (нестабильно, юридически спорно, требует постоянной поддержки при любом изменении разметки).

### Итоговая рекомендация
- Реалистично покрыть **только Melbet** через платный тариф odds-api.io (от £99/мес).
- Для «Зенит», Fonbet, Winline готового официального API не нашёл — предлагаю либо (а) исключить их из обязательной проверки и оставить существующий The Odds API как основной источник, либо (б) если у вас есть свой источник/партнёрский фид по этим трём БК (например, доступ от самих букмекеров или B2B-партнёра), интегрировать его отдельно.

## 5. Как это отразится на карточке прогноза (когда источник подключён)

После подключения источника, покрывающего нужные БК, карточка прогноза должна:
- показывать по каждой из 4 БК отдельно: событие+рынок найден/не найден, и если найден — реальный коэффициент;
- показывать лучший (максимальный) коэффициент среди найденных;
- **не включать** событие в основные рекомендации, если рынок не найден ни в одной из 4 БК.

Это можно реализовать сразу после того, как определимся с источником(ами) — правок в `candidate_builder.py`/карточке `selection_engine/report.py` не потребует много времени, но зависит от того, какие БК реально будут покрыты.
